## Executive Summary

论文提出 EPC 框架用于测量自适应 LLM 代理中评估器驱动的偏好动态，并报告多评估器审计发现 GPT-4o 版本漂移导致偏好耦合反转。最大亮点是围绕版本不稳定性的元诊断思想，且大量实验数据与代码公开。但致命问题有三：(1) 通篇未与任何已发表的评估器偏好或漂移度量方法进行对比，无法证明 EPC 相较现有工具的诊断优势；(2) 输出格式与主指标 PCI 高度混淆（聚合级ρ≈0.89），而论文仅靠不显著的实例级检验便宣告排放，却未提供长度归一化或受控消融，使测量有效性严重存疑；(3) 全部普适性结论停留于单个 GPT-4o 在单一时间窗口内的版本漂移案例，缺乏系统性的多模型、多时间点追踪，亦无与人类判断或下游任务结果的关联。这些缺陷动摇了框架声称的诊断能力、实证效度和推广性，整体工作尚未达到顶会接收标准。

## Detailed Assessment

### Major Strengths
- 将多评估器、多条件、多版本的审计统一在同一个诊断框架下，系统性地揭示了闭环自适应当中评估器偏好不稳定的现象，为社区敲响警钟。
- 完整公开了 122 次实验的 JSON 数据、代码和参数，具有良好的可复现性基础，便于他人检验和复用。
- 通过无基线消融和对称学习率再实验等技术手段，主动排除了部分协议伪影，显示了作者对内部效度的关注。

### Major Concerns
- 基线对比完全缺失：全文唯一“基线”均由自身实验变体或随机水平构成，从未与 reward overoptimization 指标、sycophancy 检测、LLM-as-judge 偏差度量等现有方法进行横向比较。这使 EPC 框架的诊断能力无从量化，声称的“标准诊断”沦为空洞主张。
- 输出格式严重混淆且未受控制：聚合级输出长度与 PCI 高度相关（ρ≈0.89），虽然实例级 p=0.093 未达显著，但样本量小且未报告效应量；且论文未采取长度归一化、协方差调整等任何纠正措施。这意味着 PCI 很可能大量捕获格式偏好，而非决策多样性，直接侵蚀框架的核心解释力。
- 实证基础单薄，普适性未建立：全部重大结论依赖 GPT-4o 从五月到六月的一次版本漂移，既无多模型、多时间点的序列证据，也未通过外部质量基准（如人类评估）验证。将个别 API 行为放大为“单一快照研究不可靠”的普遍规律，属于过度宣称。

### Minor Issues
- 耦合系数 γ 无界，对权重向量的微小扰动高度敏感，论文未讨论极端情形下的稳定性，亦未与有界替代指标（如仅用 JSD）做系统比较。
- 表格 5 中的符号解释不全，MPCI 公式括号歧义，摘要与正文对指标层次的表述不一致，降低了表达精确度。
- 部分辅助分析（如 PCI-Accuracy 校准仅有 N=10）统计效力不足，削弱了关于自评估稳定性的推断。

## Revision Suggestions

1. 必须与至少两个已发表的评估器偏好/漂移度量方法（例如 reward overoptimization 量化指标、sycophancy 检测框架、LLM-as-judge 偏差分数）在相同实验条件下做横向对比，从检测灵敏度、误报率、早期预警能力等维度量化 EPC 的优势。
2. 必须解决输出格式混淆。具体措施包括但不限于：(a) 训练长度受控生成或实施输出长度截断/归一化后重新计算 PCI；(b) 将输出长度作为协变量进行统计调整，并报告调整前后的指标变化；(c) 设计消融实验刻意解耦长度与策略，以证明 PCI 仍能捕捉策略多样性。
3. 必须大幅扩展实验的任务集和执行器多样性。至少包含数十个覆盖推理、知识、安全、多模态的真实任务，并使用至少两个不同架构的执行器，以评估结论的普适性。
4. 必须提供多模型、多时间点的版本漂移证据，而非仅靠一次 GPT-4o 快照。建议连续记录至少 3 个月，涵盖多个主流模型系列，并结合提供商更新日志做关联分析，将“警告”从轶事升级为可重复的实证规律。
5. 必须建立框架与外部质量信号的关联，例如引入人类判断或下游基准性能，以验证 EPC 指标在实际有害性上的诊断价值，否则“偏好坍塌”的严重性无从衡量。
6. 必须论证耦合系数 γ 的统计行为，或者将其退化为辅助指标，改用有界、稳健的 JSD 作为主度量，并提供灵敏度分析以证明所设阈值具有跨条件的适用性。
7. 在涉及多重条件比较时，必须采用恰当的多重检验校正（如 Bonferroni 或 FDR），并系统报告所有关键分析的效应量和置信区间。
8. 修正摘要与正文之间的指标层次不一致，消除 MPCI 公式中的括号歧义，并补全表格符号含义。

## Criterion-by-Criterion Analysis

### baseline_comparison (Weight: 10%, Score: 1)
该项是论文最严重的短板。作者声称构建了“诊断框架”，却回避与任何已发表的评估器动态度量方法比较。报告中列出的所谓基线全为自身变体或随机水平，未引入 reward overoptimization、sycophancy 检测或 LLM-as-judge 偏差度量等竞品。这使得 EPC 在检测灵敏度、误报率、预警能力等关键维度上的性能完全未知。鉴于论文的核心贡献正是“测量基础设施”，缺乏外部基准直接导致其贡献无法定位，我完全同意审稿人的 1 分评价。

### experimental_design (Weight: 16%, Score: 3)
实验在版本漂移检测和消融设计上表现出一定严谨性，但任务集过小（仅 8 个文本任务和 8 个文本代理视觉任务）、执行器单一，且输出格式混淆未被有效排除（实例级 p=0.093，判定不显著却未控制长度）。此外，所有动态均基于评估器自身输出，从未与人类判断或下游任务性能关联。这些缺陷限制了结论的普适性和解释力，审稿人 3 分评价适当，但格式混淆问题在本裁决中被视为更严重的致命伤。

### impact_significance (Weight: 8%, Score: 2)
本文确实指向了一个真实且被低估的风险——模型静默更新可导致闭环自适系统结果逆转——但影响被狭窄的实证基础拖累。发现仅基于一次 GPT-4o 漂移，且由非官方 API 代理完成，未来行为不可预测。缺乏多模型、长时间窗口的追踪，使“警告”的价值停留在轶事层面。审稿人评价 2 分，我认可。若实验基础得到根本加强，其影响力可大幅提升，但当前状态不足以产生领域范围的推动。

### originality_novelty (Weight: 15%, Score: 2)
EPC 框架的主要指标（PCI、JSD、耦合矩阵）均为已有工作或简单派生，缺乏实质性的方法论创新。版本漂移本身属于经验发现而非分析技术，且同期已有工作处理漂移检测，本文仅测量了其下游效应。整体贡献更接近己有工具的工程化拼装与案例收集，距领域前 10% 的突破性工作尚有显著距离。审稿人 2 分评价较为中肯。

### presentation_clarity (Weight: 9%, Score: 4)
论文组织结构清晰，多数术语有定义，且坦诚讨论了策略认知验证缺失等局限，值得肯定。存在的瑕疵包括摘要与正文指标层次不一致、MPCI 公式括号歧义、诊断表符号说明不完整等，但均属可快速修正的表达问题，不影响整体理解。审稿人 4 分评价合理。

### reproducibility (Weight: 12%, Score: 4)
作者提供了全套代码、122 次实验的 JSON 数据和详细超参数，可复现性基础坚实。依赖商业 API 和第三方代理导致精确复现受限，但论文透明记录了版本漂移现象，反而体现了诚实。真实图像实验细节不足是次要瑕疵。审稿人 4 分评价适当。

### statistical_robustness (Weight: 12%, Score: 3)
核心发现使用了 bootstrap、permutation 检验和强效应量（d=4.37），显著性可靠。但多重条件比较未校正，部分辅助分析样本量过小且缺少效应量与置信区间，γ 的无界性未通过模拟评估。整体稳健性受到一些损伤，审稿人 3 分评价合理。

### technical_soundness (Weight: 18%, Score: 2)
技术正确性的主要威胁来自输出格式混淆未解决，以及 γ 指标的无界性与敏感性。PCI 高度关联输出长度，使“偏好坍塌”可能实质上是格式趋同，而非策略收缩。协作者虽进行了实例级检验，却未采取任何校正措施，令大量后续解读根基不稳。加之单一版本漂移的过度推广和校准实验的不充分，技术扎实程度大打折扣。审稿人 2 分评价可接受，但格式混淆在这里应视为接近致命的问题。

## Technical Discussion

论文最具争议的策略是选择用输出长度高度相关的指标（PCI）作为偏好动态的主度量，却仅通过一次统计不显著的实例级检验（p=0.093）就声称格式混淆被排除。即使实例级不显著，聚合级强相关清楚地表明：当执行器产生更长的回答时，其权重向量便趋同，从而推高 PCI。那么，EPC 究竟是在测量推理策略的变化，还是在测量输出啰嗦程度的波动？论文并未设计长度归一化汇报、受控长度生成或协变量调整等实验来切割这一混淆，此漏洞直接挑战了所有基于 PCI 的发现。除此之外，依赖单次 API 漂移得出“任何快照研究不可靠”的结论，相当于用一个采样点推断整体气候——缺少多模型、多时间窗口的时序数据，因果归因为零。框架本应承担诊断职责，却未与任何已有诊断工具做 A/B 对比，则在方法论上留下了另一重空白。这些问题不是边缘修补能够解决的，需要从实验设计到指标定义的重新构筑。
